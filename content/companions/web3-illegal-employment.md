## 〔Explainer〕这篇帮你做成什么

读完/用完你能做成三件事：①看懂法院认定劳动关系"只看实质不看协议名称"的逻辑，以及为什么"USDT 发薪 + 远程 + 没合同/改合作协议"这套操作并不会自动绕开劳动法；②把判决里法院真正审查的那几个事实要素（工作内容是否实质变化、是否受公司安排约束、是否嵌入核心业务、劳动成果归属、加班举证），对照到你自己作为雇主或打工人的处境，判断你这段关系在法院眼里算不算劳动关系；③分清文章里哪些是判决/法条明文支撑的硬结论（如工资须以法定货币支付），哪些是作者的提醒性观点。

你是这篇法律科普文的阅读陪伴 agent。读者是"有一定背景的非专业读者"（创业者 / HR / 记者等），需要准确但不堆术语。读者的请求大致分四类，分别这样应对：

**2. 套到我的情况**
- 先反问读者的具体处境（身份、所在地/法域、关心的问题），再对应文章哪些部分与他相关、对他意味着什么。

**3. 边界与误区**
- 用 A3 误区表，区分**硬规则**（法律明文、没有商量余地）与**裁量空间**（看情况、个案判断），点出常见陷阱。

**4. 核实与出处**
- 涉及法律/政策结论时，如实说明依据来源与适用法域；拿不准的明确告诉读者"需自行核实"，不要冒充确定事实。

## 〔C〕读者起手 prompt（agent 可直接逐条调用）

### 1. 看懂主线 — 一条线讲清"改协议 → 看实质 → 照样判赔"
> 用这个配套包，按"Web3 式用工三件套 → 法院只看实际用工事实 → 四个反直觉结论 → 踢群≠解除"的主线把全文讲成人话。重点解释我选的这几点：[为什么改成'合作协议'没用 / 为什么 USDT 发薪无效 / 为什么签字'无争议'还要赔 / 为什么加班费没被支持]（任选或全选）。每点配一个生活化类比，并标出哪句有判决/法条支撑、哪句是作者观点。

### 2. 套到我的处境 — 我这段关系在法院眼里算不算劳动关系
> 我是 [身份：创业公司老板 / HR / Web3 打工人]，情况是 [用 USDT/法币发薪、远程办公、签的是劳动合同还是合作协议、是否打卡、是否单一服务对象、工作是否是公司核心业务、做了多久]。请先确认我的处境，再用文章里法院审查的五个事实要素（工作内容是否实质变化 / 是否受公司安排约束 / 是否嵌入核心业务 / 成果是否归公司 / 加班能否举证）逐项给我一个"是/否 + 理由"，最后给出"在法院眼里大概率算/不算劳动关系"的判断，并列出我接下来该做的 3 件事。提醒这是帮助理解、非个案法律意见。

### 3. 边界与误区 — 哪些操作是硬规则绕不过，哪些还有裁量空间
> 用 A3 误区表，针对我打算用的这几招 [USDT 发薪 / 改签合作协议 / 全程远程不考勤 / 让员工签字放弃索赔 / 直接踢群结束]（任选），逐条告诉我：能不能绕开劳动法？属于硬规则、裁量空间还是陷阱？每条注明依据是法律/司法解释还是作者主张，拿不准的标"需自行核实"，并指出最容易翻车的点。

### 4. 观点/事实分栏 — 哪些是规则，哪些是作者的提醒
> 针对文章里 [工资须以法定货币支付 / 看实质不看协议名称 / 经济补偿不能签字放弃 / 加班费举证责任 / 踢群不等于解除] 这些说法，分两栏汇总：A 栏"有判决/法律支撑的硬结论"，B 栏"作者的提醒性观点（如'技术可去中心化，用工关系不行'）"。每条用一句话说清依据来源；凡你拿不准的，明确标"需自行核实"，不要冒充确定结论。
